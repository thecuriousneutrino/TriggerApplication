23 ### Vertex distribution
24 
25 ```
26 n_vertices_mean N
27 
28 x_mean_pos POS
29 x_width WIDTH
30 y_mean_pos POS
31 y_width WIDTH
32 z_mean_pos POS
33 z_width WIDTH
34 
35 max_z_pos FLOAT
36 max_r_pos FLOAT
37 
38 flat_r BOOL
39 ```
40 
41 Specify the simulated vertex distribution to generate.
42 
43 In order to create a uniform distribution, set `max_z_pos` and `max_r_pos` to the size of your detector, and `x_width`, `y_width`, `z_width` all to negative values
44 * `n_vertices_mean` Poisson mean of the number of vertices to generate
45 * `x_mean_pos` The mean `x` position (cm)
46 * `x_width` The width of the Gaussian in `x (cm)`
47  * If 0, no throws
48  * If <0, throw uniformly between `±max_r_pos`. In this case, `x_mean_pos` is ignored
49 * `y_mean_pos` The mean `y` position (cm)
50 * `y_width` The width of the Gaussian in `y` (cm)
51  * If `0`, no throws
52  * If `<1E-6`, throw uniformly between `±max_r_pos`. In this case, `y_mean_pos` is ignored
53 * `z_mean_pos` The mean `z` position (cm)
54 * `z_width` The width of the Gaussian in `z` (cm)
55  * If 0, no throws
56  * If <0, throw uniformly between `±max_z_pos`. In this case, `z_mean_pos` is ignored
57 * `max_r_pos` Events with larger `r` (in cm) than this will be rethrown
58 * `max_z_pos` Events with larger `|z|` (in cm) than this will be rethrown
59  * Note that this is a detector half-height because the co-ordinate origin is at the detector centre
60 * `flat_r` If this is true, and both `x_width` and `y_width` are negative (i.e. generating a random distribution in the circularly plane), use this to generate a flat distribution in `r` (rather than flat in `x` and `y`)
